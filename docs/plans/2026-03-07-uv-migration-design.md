# Design: Migrate to uv

**Date:** 2026-03-07

## Goal

Replace legacy `setup.py` + `setup.cfg` + `requirements.txt` with a modern `pyproject.toml` and use `uv` as the package manager for dependency resolution, local development, and CI.

## Files Changed

| File | Action |
|---|---|
| `setup.py` | Delete |
| `setup.cfg` | Delete (bumpversion config moves to `pyproject.toml`) |
| `requirements.txt` | Delete (deps move to `pyproject.toml`) |
| `pyproject.toml` | Create |
| `uv.lock` | Create (generated by `uv lock`) |
| `.github/workflows/python.yml` | Update |

## pyproject.toml Structure

- **Build backend**: `setuptools` with `find_namespace_packages` to preserve the `sphinxcontrib` namespace package
- **Project metadata**: name, version, description, license, author, classifiers, URLs
- **Dependencies**: Sphinx, docutils (previously in `requirements.txt`)
- **Optional deps**:
  - `[docs]`: Sphinx, myst-parser (previously in `docs/requirements.txt`)
  - `[dev]`: pytest
- **Bumpversion**: `[tool.bumpversion]` section (replaces `[bumpversion]` in `setup.cfg`)

## CI Update

- Use `astral-sh/setup-uv@v5` (replaces `actions/setup-python` + pip)
- `uv sync --extra dev` to install package and dev dependencies
- `uv run pytest` to run tests
- Python matrix updated to `["3.10", "3.11", "3.12"]`

## Publishing to PyPI

```bash
uv build     # builds sdist and wheel
uv publish   # uploads to PyPI (use --token or PYPI_TOKEN env var)
```

## Decisions

- Keep `docs/requirements.txt` as-is for the docs build Makefile/script (avoids changing the docs build workflow)
- Use setuptools backend (not hatchling) to avoid namespace package compatibility issues
